Getting Started

For system security related challenges,

You will see a server endpoint nc xxx.xxx.xxx.xxx yyyy and an attached zip file.

The attached zip file will contain

  1. The source code of the program
  2. Compiled binary of the program
  3. Makefile to compile the program

The server is hosting the compiled binary at the endpoint. Your goal is to exploit the program and get the flag out from the server.

To run the program, just type

./<binary_file>

If you encounter permission issues, remember to do

chmod +x <binary_file>

GDB and Pwntools

While doing system security related challenges, you might face the following problems.

  1. How do I get an address of a function?
  2. How do I inspect the stack during runtime?
  3. How do I inspect the registers during runtime?
  4. How do I test my exploit locally?
  5. How do I construct a payload and send it to the server?

Here, we will provide a tutorial and some tips to solve the stated problems using GDB.

GDB is a GNU debugger that is very useful for developers (or hackers) to perform runtime analysis of a program.

However, the default GDB program is not really useful for us hackers and it is hard to use. Fortunately, there are many GDB plugins that can make a hacker’s life easier.

For this assignment, we recommend pwndbg. You can also use other plugins like peda or gef

pwndbg Installation

To install gdb with pwndbg, do

sudo apt-get update
sudo apt-get install gdb
git clone https://github.com/pwndbg/pwndbg
cd pwndbg
./setup.sh

Verify that you have succesfully installed gdb by running gdb on the command line, you should see something like this:

If you encounter any installation issue, you can contact the TAs for help.

M1 Mac Installation Issue

Note that M1 Mac does not work well with gdb. If you are using an M1 Mac, then you need to find another computer that runs x86_64. You can either choose to use raw GDB on sunfire or consider starting a cloud ubuntu instance and install GDB there.

For cloud instances, you can use google cloud free tier or free credit to start a compute engine.

Basic usage of GDB

We will walkthrough the basic usages of GDB with the challenge bouncer

To attach GDB to a binary program, just run

gdb bouncer.bin

You can now start the program with the start command. The start command will insert a breakpoint at the start of the program. The program will be executed and stopped immediately before the first line of code

pwndbg> start

Now you will see a beautiful screen. Don’t be afraid as we will explain each componnent in detail.

Registers

The first part shows the value of all registers. The -> arrow means address -> value.

For example,
RDX stores the value 0x7fffffffe418 and the value stored in 0x7fffffffe418 is 0x7fffffffe6c2. In C language, this means

*(0x7fffffffe418) = 0x7fffffffe6c2

The <- arrow gives you a preview of the value.

Taking the same RDX example,

0x7fffffffe6c2 stores the string 'SHELL=/bin/bash'

Assembly

The second part is the disassemble of the main function.

Take the first line as example,

0x5555555551f8 is the address of the assembly code, <main + 8> means that the corresponding address is 8 bytes from the start of the main function, sub rsp, 0x30 is the assembly code itself.

Stack

Note that the stack grows from higher addresses to lower addresses. So, you will see that the top address in the stack is of a lower value, whereas the bottom is of a higher value.

rbp -> base pointer register
rsp -> stack pointer register

Commands

Now the program is paused, how do you continue the execution? Here are some useful commands that can help you.

start -> Break immediately before the start of the program
n -> Run the next line
s -> Step into the next line
c -> Continue the program until the next breakpoint
r -> Run the program until it blocks or exits
b *addr -> Add a new breakpoint at addr
stack 30 ->  View the top 30 address store in the stack
disass func -> Disassemble the function func
tele addr -> View the content at addr
info address func -> Find the address of func
vmmap -> View the memory layout of the program

The commands above should be sufficient for this assigment. If you wish to know more about GDB. You can refer to these guides (or just Google): https://tc.gts3.org/cs6265/2020/tut/tut02-warmup2.html and https://parzelsec.de/posts/speed-up-your-exploits

Pwntools for system security

As mentioned in the Python3 cheatsheet, pwntools is a very helpful tool to send payloads to the server.

The best thing about pwntools is that it can send raw bytes!

There are some methods in pwntools which are really useful for system security.

p64(x) -> Convert an integer x to 8 bytes little endian
p32(x) -> Convert an integer x to 4 bytes little endian
process("some_file") -> Run "some_file" locally

Example of pwntools usage

from pwn import *

p = process("./x.bin") // Replace this with remote to connect to server

addr = 0x400367

payload = b""
payload += b'A' * 40 // Fill in buffer
payload += p64(addr) // Convert addr to bytes little endian

p.sendline(payload)

p.interactive() // Switch to interactive mode
